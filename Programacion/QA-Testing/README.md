# QA y Testing

Este módulo enseña a planificar y verificar calidad de software en proyectos reales, con especial atención a equipos pequeños que construyen frontend, backend y aplicaciones móviles. La meta no es automatizar todo: es detectar riesgos temprano, proteger los flujos importantes y mantener pruebas que den confianza al publicar.

El contenido combina pruebas manuales, exploratorias y automatizadas. También conecta calidad con API, bases de datos, CI/CD, accesibilidad, seguridad básica y retroalimentación del equipo.

## Ruta del módulo

1. Fundamentos y estrategia de calidad.
2. Diseño y gestión de pruebas manuales.
3. Niveles y tipos de prueba.
4. Automatización pragmática.
5. Calidad en el ciclo de entrega.
6. Proyecto aplicado y portafolio.

El archivo de lectura compatible con Lectura Lenta está en `sources/contenido-completo.md`. El proyecto práctico está en `proyecto-qa.md`.
